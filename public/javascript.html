Javascript

Type coercion

Type coercion is implicit type conversion, as opposed to type casting which is explicit type conversion.

Type coercion is best avoided by using the strict equality operators such as === and !==

Scope and Const

Often, the more static/constant you make code, the easier it is to reason about. As opposed to it being unneccesary dynamic/varying.

Therefore it is best to use const when possible.

It is best to restrict variables to only be available in the scope in which you intend for them to be used. If variables have an unncessary large/wide scope, the code becomes more difficult to reason about.

Functional programming

Functional programming is programming where you use pure functions that dont have any side effects.

If functions have side effects, they often become more difficult to reason about when composed together. If you on the other hand use pure functions, then you know exactly what the inputs and outputs are at all times. That is, pure functions are often more composable.

SSR (server-side-rendering) vs. CSR (client-side-rendering.)

The advantages of SSR depends a lot on how static the content is.

If the content served is the same for all users, then you can render the content once, cache it on the server, and then serve it to all users (including search engines). So SSR is great for content where you want to do SEO.

On the other hand, if the content is different for each user, then the server needs to render it for each user. In that scenario, for scalability reasons, then it might be preferred for the rendering to be done on each users own computer. The content is also unlikely to be a target of SEO if the content is different for each user.

The reason that SSR is better for SEO is that most crawlers do not support javascript and CSR.

Another consideration to have in mind when comparing SSR and CSR is what the latency is between the client and the server. If the latency is very high, then it might be a good idea to explore if data can be sent speculatively to the client ahead-of-time, which is then rendered by client.
