node.js

node.js is a javascript runtime environment where the browser APIs have been removed and replaced with APIs suitable for server-side software.

node.js is based on the same javascript virtual machine used in chrome.

you can run a node app named app.js like this:

node app.js

You can specify third-party dependencies in a package.json file like this:

{
    "dependencies": {
        "express": "^4.18.2"
    }
}

And then run:

npm install

In order to install all the dependencies.

nodemon

nodemon monitors for file changes and automatically restarts node upon any changes.

if there is a package.json file, then nodemon uses "main" from package.json in order to determine which file should be passed to node.

nodemon can be configured with a nodemon.js config file.

nodemon can be made to monitor for file changes in files other than .js files by using the "--ext" option.

express

express is a node module that provides routing and "middleware" functions.

express only contains very few built-in "middleware" functions.

"middleware" functions are just callback functions which are called in sequence in response to a specific route.

The advantage of using express is that there are a lot of third-party node modules that provide express compatible "middleware" functions.

The following is an example of a minimal node app that uses express:

const express = require("express");
const app = express();

app.get("/", (req, res) => {
    res.sendFile(__dirname + "/public/home.html");
});

app.listen(8080);
